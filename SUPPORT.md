# Support

PoseCap support starts with reproducible diagnostics so an animator can recover
quickly and maintainers can distinguish setup, hardware, Blender, and engine failures.

## Installation and usage

1. Follow the [Getting Started guide](doc/guides/getting-started.md) and the focused
   guides linked from the [README](README.md#get-started).
2. Run PoseCap Doctor and address every reported error.
3. Create a Support Bundle from the PoseCap panel and review its summary and files.
4. If the problem remains, open a [bug report](https://github.com/CorridorTech/PoseCap/issues/new?template=bug.yml)
   with the requested build, Windows, Blender, GPU, reproduction, and diagnostics.

Never publish passwords, browser cookies, licensed model files, private footage, or
other people's data. A Support Bundle is designed to exclude credentials, but the
sender remains responsible for reviewing what they share.

## Security and conduct

Use a [private security report](https://github.com/CorridorTech/PoseCap/security/advisories/new)
for vulnerabilities. Do not use a public issue for suspected credential exposure,
arbitrary code execution, unsafe archive handling, or another exploitable condition.

Community participation follows [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Scope

The issue tracker covers reproducible PoseCap defects and feature outcomes. General
Blender instruction, GPU driver administration, third-party model licensing, and
custom character-production work remain with their respective vendors or production
teams unless PoseCap itself is the failing boundary.
